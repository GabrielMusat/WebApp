What is

Artificial Intelligence?

"The theory and development of computer systems able to perform tasks normally requiring human intelligence" ~ Sciencedirect

Examples include:

  • Speech recognition
  • Decision-making
  • Language translation

In this case, we'll be looking into "Deep learning models". A very common example of a deep learning model+ would be AI chatbots, such as ChatGPT. These deep learning models can, and do help companies to protect their users' data. Currently, one of the best examples of one of these companies would be IBM, which uses AI to: "accelerate threat detection expediting responses, and protecting user identiy and datasets".1 Later on, we will look at more of these examples in detail. However, to get back to the point, these chatbots aren't only positive, because hackers can also profit off of AI, using various tactics which we will also discuss later, to obtain code which is way more efficient in hacking into companies an databses.

Pro's and Con's

(of AI's use in cybersecurity)

Pro's

  • AI is mainly only limited by the speed of it's code, allowing it to analyze 1000's of lines of code and find errors in only a few seconds, while it would take a human days.
  • Because AI is so fast at analyzing, it can also give tips really quickly on how to fix a problem, or even edit the code itself to make it more secure. One way the AI does this is trying to hack in mock servers with the companies security protocols, and document on how it managed, as well as possible solutions. However currently companies like Microsoft still use humans for this job.
  • AI has the capability to learn and use previous mistakes and to learn from them and find patterns. The AI can think on it's own, and is able t do more thatn just following instructions. This makes security easier for companies, because they do not have to code every scenario and counter
  • Because AI is adaptable, it is also able to take action much faster compared to pure code, as it doesn't have to ask for permission from certain rules, but rather can take action on it's own.
  • One last way on how companies can use AI, is to create an AI, and fake servers with their security protocols, and have the AI try to hack into the servers, and document on how it managed, as well as possible solutions.

Con's

  • Hackers can use AI to obtain malicious code in a few ways, one of them being: Adverserial attacks on AI systems, meaning writing inputs designed specifically to force the AI to generate malicious code against it's own policy.
  • Another way would be: Data poisoning. Which is where you give a AI biased or wrong datasets in order to get biased answers or code which isn't true, and otherwise would be unobtainable.
  • Hackers can create their own AI which does the opposite of what companies use it for. It creates scenarios, which the hacker tries to hack into, and give the hacker tips, however a lot of expertise is needed
  • Finally, another way AI can be manipualted is for hackers to hack into the actual AI of a company, and set up so-called "back doors", as an entrance if they get kicked out. They can then use the information the AI gathered for the company in terms of security weaknesses and use it against the company. This is why most companies still use people to hack into servers in a controlled manner, and not an AI.
